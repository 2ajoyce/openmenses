# Onboarding Flows — opencycle

This document describes the intended onboarding experience for new users.

> **Note:** This is a placeholder. Detailed flows will be defined when UI development begins.

## Goals

- Get the user tracking their first cycle with minimal friction.
- Clearly communicate that all data stays on their device.
- Require no account creation or network connection.

## Planned Flows

### First Launch
1. Welcome screen — introduce the app and privacy promise.
2. Optional: brief explanation of what the app tracks.
3. Prompt to log their last period start date (or skip).
4. Home screen.
